# Resource Feed AYX

本仓库用于归档和发布多个独立的 HTML 页面。这些页面作为静态资源存放，不针对任何特定域名或网站。

## 项目简介

这是一个资源归档性质的仓库，主要用途是存放各类独立的 HTML 文件。页面内容可能涉及信息展示、工具页面或其他静态呈现形式。所有页面均为独立文件，不依赖后端服务。

## 目录说明

```
.
├── index.html          # 仓库主页面（可选）
├── pages/              # 存放主要 HTML 页面
│   ├── page-1.html
│   ├── page-2.html
│   └── ...
├── assets/             # 公共资源文件（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `pages/`：所有独立 HTML 页面存放于此目录，每个文件对应一个独立页面。
- `assets/`：页面共用的样式、脚本或图片资源，按类型分文件夹存放。

## 页面归档说明

- 每个 HTML 页面均为完整独立的文档，可单独打开浏览。
- 页面之间不存在交叉引用或依赖关系。
- 资源文件（CSS、JS、图片）统一放在 `assets/` 目录下，页面通过相对路径引用。
- 归档内容可能不定期更新或调整，不保证所有页面长期可用。

## 维护说明

- 本仓库由维护者不定期更新，新增或修改页面时会同步更新目录结构。
- 如发现页面显示异常或链接失效，可提交 Issue 反馈。
- 不接受外部直接提交的页面内容，但欢迎通过 Issue 提出建议。
- 仓库仅用于归档目的，不提供任何形式的服务承诺。
